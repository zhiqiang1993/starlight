What is a Content Delivery Network (CDN)?

A Content Delivery Network (CDN) is a globally distributed network of servers that work together to provide fast delivery of Internet content including web pages, images, videos, and other assets.

By caching content at strategically placed physical nodes across the globe, a CDN allows for the quick transfer of assets needed for loading Internet content.

How Akamai CDN Works

      │                                  │                              │
  1. User requests content - When a user requests a web page or asset, the request is routed to the nearest Akamai edge server instead of directly to the origin server.
  2. Edge server check - The edge server checks if it has a cached copy of the requested content that is still valid according to its TTL (Time To Live).
  3. Cache hit - If the content is in cache and valid (a "cache hit"), the edge server delivers it directly to the user without contacting the origin server.
  4. Cache miss - If the content is not in cache or has expired (a "cache miss"), the edge server requests it from the origin server, caches a copy, and then delivers it to the user.

Key Akamai CDN Features

Content Caching

Stores copies of content at edge servers to reduce load on origin servers and decrease content delivery time.

Dynamic Content Acceleration

Optimizes the delivery of dynamic, non-cacheable content through route optimization and connection optimization.

Image & Video Optimization

Automatically optimizes images and video for different devices and connection speeds.

Web Application Firewall

Protects websites from common web exploits that could affect availability, compromise security, or consume excessive resources.

DDoS Protection

Provides defense against Distributed Denial of Service attacks by absorbing and filtering malicious traffic.

Bot Management

Identifies and manages automated bot traffic to protect against scraping, credential stuffing, and other automated attacks.

Akamai CDN Configuration Basics

Configuring an Akamai CDN involves several key components:

  • Property Configuration - Defines how content should be handled, cached, and delivered.
  • Cache TTL Settings - Determines how long content remains valid in the cache.
  • Origin Settings - Specifies the origin server(s) from which the CDN retrieves content.
  • Security Settings - Configures WAF rules, rate limiting, and other security features.
  • Content Purging - Mechanisms to remove content from the cache when needed.

This test website can be used to experiment with these settings and observe their effects on content delivery.
